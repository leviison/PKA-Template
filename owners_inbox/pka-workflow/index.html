Brief Lifecycle

A brief exists as a temporary markdown file only while work is in progress — the database row is the permanent record that survives after the file is deleted.

flowchart TD START(["Levi requests work"]) START --> LEROY_W["Leroy writes\nbrief_ref.md → Team Comms/"] LEROY_W --> DB_INSERT["INSERT INTO briefs\nstatus open"] DB_INSERT --> TEAM_READ["Team member reads brief file\nfrom filesystem — NOT from conversation\nContext isolation preserved"] TEAM_READ --> WORK["Team member executes work"] WORK --> DELIVER["Deliverable written to\nOwners Inbox/"] DELIVER --> DB_DELIV["INSERT INTO deliverables\nbriefs.id FK + file_path"] DB_DELIV --> FILE_DELETE["brief_ref.md DELETED\nfrom Team Comms/"] FILE_DELETE --> DB_UPDATE["UPDATE briefs SET\nstatus complete\ncompleted_at set"] DB_UPDATE --> FEEDBACK["Levi reviews deliverable\nand provides rating"] FEEDBACK --> DB_FEED["INSERT INTO feedback\nrating · notes · brief_ref"] DB_FEED --> DONE(["Work cycle complete"]) FILE_DELETE -.->|"FILE IS TRANSIENT\ndeleted after use"| NOTE_FILE(["🗑️ File gone"]) DB_UPDATE -.->|"DB RECORD IS PERMANENT\nsurvives forever"| NOTE_DB(["🗄️ Row preserved"])

Database Schema

The full pka.db entity-relationship diagram — assets feed content, briefs produce deliverables which collect feedback, with journal, knowledge, and settings as standalone tables.

erDiagram assets { int id PK string filename string path string type string status } content { int id PK int asset_id FK string content_type string body } briefs { int id PK string brief_ref string assigned_to string status string completed_at } deliverables { int id PK int brief_id FK string file_path string created_by string created_at } feedback { int id PK string brief_ref string team_member int rating string notes string created_at } journal { int id PK string date string title string body string tags } knowledge { int id PK string title string body string tags } settings { string key PK string value } assets ||--o{ content : "extracted into" briefs ||--o{ deliverables : "produces" deliverables ||--o{ feedback : "rated by"
